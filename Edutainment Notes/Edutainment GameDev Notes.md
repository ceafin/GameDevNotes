# Number/Word Munchers
## Mechanics

- **Grid movement** — discrete cardinal movement on a 5×6 cell grid, input-locked to one cell per keypress
- **Eat action** — validate current cell content against the round's rule; correct = score + clear cell, incorrect = lose life
- **Round rule system** — each round declares a rule (e.g., "primes", "factors of 12", "short vowel sounds"); all cells evaluated against it
- **Cell content generation** — populate grid with a mix of valid and invalid items per rule, ensuring solvability
- **Troggle AI** — enemies enter from edges, move on the grid on a timer, kill player on overlap; 2–3 behavior types (random walk, patrol, chase)
- **Troggle spawn schedule** — escalating frequency/count tied to round number
- **Round progression** — eat all valid cells → brief Troggle cutscene → new grid with harder rule
- **Life system** — 3 lives, lost by eating wrong answer or Troggle collision
- **Bonus round** — periodic reward scene between rounds (no enemies, timed point grab)
- **Score display** — running score with per-eat points scaled by round difficulty

## Data Structures

- **`Grid`** — `Array[Array[CellData]]` (5 rows × 6 columns)
- **`CellData`** — `{ content: String, is_valid: bool, is_eaten: bool }`
- **`RoundRule`** — `{ rule_type: enum, parameter: Variant, prompt_text: String, validator: Callable }` — e.g., `{ MULTIPLES, 7, "Multiples of 7", func(n): return n % 7 == 0 }`
- **`RuleDeck`** — `Array[RoundRule]` ordered by difficulty, drawn sequentially
- **`PlayerState`** — `{ grid_pos: Vector2i, lives: int, score: int, current_round: int }`
- **`TroggleState`** — `{ grid_pos: Vector2i, behavior: enum, move_timer: float, active: bool }`
- **`ContentPool`** — dictionary/resource per domain: `{ domain: "math"|"language", entries: Array[String|int] }` — source data for cell generation

## Godot Node Structure

```ascii
GameScene (Node2D)
├── Grid (TileMapLayer or GridContainer)
│   └── CellNodes × 30 (Label or RichTextLabel per cell)
├── Player (Sprite2D) — snaps to grid positions
├── Troggles (Node2D)
│   └── Troggle × N (Sprite2D + Timer for movement ticks)
├── HUD (CanvasLayer)
│   ├── ScoreLabel
│   ├── LivesDisplay
│   └── RulePrompt (Label — "Find the multiples of 7")
└── RoundManager (Node) — owns rule deck, spawn logic, round transitions
```

## Content Swap (Number ↔ Word)

The only difference between Number Munchers and Word Munchers is the `ContentPool` and `RoundRule` set:

| |Number Munchers|Word Munchers|
|---|---|---|
|**Content type**|`int`|`String`|
|**Pool source**|Generated ranges (1–100)|Word list resource file|
|**Rule examples**|`n % param == 0`, `is_prime(n)`|`has_short_vowel(w)`, `is_noun(w)`|
|**Validator**|Arithmetic function|String/tag lookup|

Everything else — grid, movement, Troggles, scoring, round progression — is identical.
